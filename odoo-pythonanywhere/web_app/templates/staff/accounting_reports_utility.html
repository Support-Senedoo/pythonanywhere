Descriptif & mode d’emploi

But. Cet utilitaire permet d’ajouter une base au registre (dans la carte « Base et filtre »), de vérifier la connexion, de lister et filtrer les rapports comptables (account.report), d’en personnaliser une copie automatique du rapport choisi (logique Senedoo : notes « custom » retirées sur les feuilles, regroupement par compte, hiérarchie des groupes de comptes, « tout déplier » désactivé) — le rapport d’origine n’est pas modifié, ou d’en supprimer une instance (si Odoo et vos droits l’autorisent).

Plusieurs bases peuvent partager le même libellé : utilisez un identifiant distinct par base (ex. client_prod / client_test) et indiquez production ou test pour chaque entrée.

  • Nouvelle base : dans la carte ci-dessous, cliquez sur « Ajouter une nouvelle base » pour afficher le formulaire → enregistrement puis rechargement avec la base présélectionnée.
  • Choisir une base : sans libellé, Actualiser affiche toutes les bases (liste déroulante + tableau avec état de connexion). Avec un libellé, seules les bases de ce client. Cliquez sur Utiliser cette base ou choisissez dans la liste déroulante.
  • Après sélection d’une base, le voyant et les informations instance (version, expiration, édition, etc.) s’affichent lorsque la connexion Odoo réussit.
  • Les noms affichés sont en français (contexte Odoo fr_FR, surcharge possible via variable TOOLBOX_ODOO_LANG). Le filtre porte sur le libellé dans cette langue.
  • Saisissez un filtre (ex. compte, SYSCOHADA, résultat) pour réduire la liste.
  • Personnaliser : indiquez l’id du rapport source (standard ou existant), tapez OUI — une copie est créée dans Odoo puis seule cette copie est personnalisée ; la barre de progression s’affiche pendant l’envoi du formulaire (navigation classique).
  • Renommer / supprimer : depuis la liste des rapports ; celle-ci se replie automatiquement après un clic sur « Personnaliser » (replier/déplier via le titre du bloc). Pour supprimer, recopiez exactement SUPPRIMER-<id> — irréversible si Odoo accepte.
