{% if accounting_mode == 'pl_standard' %}
Descriptif & mode d’emploi — compte de résultat personnalisé

But. Lister les account.report, puis créer une copie d’un compte de résultat (ou rapport de même famille) et appliquer la personnalisation Senedoo détail comptes / SYSCOHADA (notes « custom » retirées, regroupement par compte, hiérarchie des groupes, « tout déplier » désactivé). L’original n’est pas modifié.

Les bases sont identifiées par le nom technique Odoo (db) ; le filtre hôte d’URL regroupe les bases d’un même serveur.

  • Personnaliser : id du rapport source + OUI — copie + script personalize_syscohada_detail.
  • Liste : filtre texte sur le nom ; renommer / supprimer depuis chaque ligne.
  • Pour le budget analytique sur P&L, utilisez l’utilitaire dédié « P&L analytique / budget ».

{% elif accounting_mode == 'pl_budget' %}
Descriptif & mode d’emploi — P&L analytique et budget

But. Même principe qu’un P&L personnalisé, avec en plus l’activation de filter_analytic et filter_budget sur la copie (si les champs existent sur votre Odoo SaaS). Utile pour piloter un budget financier avec filtre analytique.

La sonde budget / analytique (lecture seule) indique si des lignes de budget portent un analytique — prérequis pour qu’un budget « par axe » ait du sens dans le rapport.

  • Lancez d’abord la sonde si besoin, puis Créer la copie (détail + analytique / budget) avec OUI.
  • Détails et contournements : DEPLOY_PYTHONANYWHERE.md.

{% elif accounting_mode == 'balance' %}
Descriptif & mode d’emploi — balance 6 colonnes

But. Partir d’une balance 4 colonnes (solde initial, débit, crédit, solde final) ou d’une balance à 2 colonnes (débit et crédit sur la période seulement), en créer une copie, puis la transformer en 6 colonnes : débit initial, crédit initial, débit période, crédit période, débit final, crédit final. L’original n’est pas modifié. En 4 colonnes, les soldes initiaux/finaux doivent utiliser la subformule sum. En 2 colonnes, les soldes sont dérivés via les champs Odoo to_beginning_of_period et from_beginning : contrôler les totaux sur une base de test ; les lignes de regroupement peuvent exiger des formules d’agrégation complémentaires.

Odoo Enterprise / SaaS (ex. v19) : la balance d’essai standard utilise un handler qui ne connaît pas les libellés de colonnes Senedoo (sn_open_deb, …) et peut provoquer une erreur RPC à l’affichage. Les versions récentes de l’outil retirent automatiquement ce handler sur la copie après transformation. Pour une copie déjà cassée : sur la fiche account.report (mode développeur), videz custom_handler_model_id / custom_handler_model_name si présents, ou refaites une nouvelle copie depuis le rapport source avec l’outil à jour.

Auteur : {{ utility_author }} · Version : {{ utility_version }} · Date : {{ utility_date }}
