Mode d’emploi — P&L analytique et budget

But. Même principe qu’un P&L personnalisé, avec en plus l’activation de filter_analytic et du filtre budgets (filter_budgets sur Odoo 17+, ou filter_budget sur d’anciennes bases) sur la copie. Documentation Balance OHADA

Rapport Balance OHADA (6 colonnes, codes bal_ohada / bal_ohada_section). Une recréation retire d’abord les instances toolbox (menus + actions client), pas les autres rapports Studio.

Hors navigateur : create_balance_6cols_via_api.py, XML, ZIP module.

Procédure manuelle Odoo (balance générale, v19)

Balance générale (pas balance d’essai Enterprise). Moteur aggregation + subformula positive pour les soldes décomposés. Toolbox : colonnes ohada6_* ; variables TOOLBOX_OHADA6_* ; boutons staff « brut » / « aggregation ».

  1. Rapport — Comptabilité → Rapports → Balance générale → modifier (droits ou mode développeur).
  2. Colonnes — retirer solde initial/final « tout en un » ; ajouter débit/crédit initiaux et finaux avec labels alignés sur les expressions (voir tableau).
  3. Ligne — groupby = account_id ; une expression par colonne (label = expression_label).
  4. Options — inclure le solde initial ; désactiver comparaison de périodes si doublons.
