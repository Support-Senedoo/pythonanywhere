But. Lister les account.report, puis créer une copie d’un compte de résultat (ou rapport de même famille) et appliquer la personnalisation Senedoo détail comptes / SYSCOHADA (notes « custom » retirées, regroupement par compte, hiérarchie des groupes, « tout déplier » désactivé).   • Personnaliser : id du rapport source, puis bouton — copie + script personalize_syscohada_detail.
  • Liste : filtre texte sur le nom ; renommer / supprimer depuis chaque ligne.
  • Pour le budget analytique sur P&L, utilisez l’utilitaire dédié « P&L analytique / budget ».

{% elif accounting_mode == 'balance' %}
Descriptif & mode d’emploi — Balance OHADA

But. Créer sur la base un rapport comptable nommé Balance OHADA (six colonnes : débit/crédit initiaux, débit/crédit période, débit/crédit finaux), sans partir d’une copie d’un rapport existant. Une nouvelle création retire d’abord uniquement les rapports créés par cette toolbox (codes de ligne bal_ohada / bal_ohada_section), avec menus et actions client associés — pas un autre rapport nommé pareil dans Studio. Suppression seule avec confirmation possible.

Repère technique : ligne feuille account.report.line avec code = bal_ohada.

Alternatives hors navigateur : script create_balance_6cols_via_api.py, gabarit XML, ou procédure manuelle ci-dessous (Studio / v19).

Procédure manuelle d’ajout des colonnes

À utiliser sur la balance générale (pas la balance d’essai handler Enterprise) pour obtenir 6 colonnes OHADA, ou pour corriger une configuration erronée (montants à zéro, « expression inconnue », mauvaise ligne). Cette procédure sert aussi de référence pour l’assistance Senedoo / l’assistant afin d’aligner les conseils avec Odoo v19.

  1. Ouvrir le bon rapport. Comptabilité → Rapports (ou Reporting selon la langue / version) → Balance générale. Icône engrenage ou menu Configuration → Modifier le rapport (selon les droits ; sinon mode développeur + Rapports comptables).
  2. Onglet Colonnes. Modèle souvent : Débit, Crédit, Solde initial, Solde final. Supprimer les colonnes Solde initial et Solde final. Ajouter quatre colonnes avec ces expression_label (identiques aux labels des expressions, étape suivante) :
    • Débit initial — debit_initial
    • Crédit initial — credit_initial
    • Débit final — debit_final
    • Crédit final — credit_final
    Conserver les colonnes période Débit / Crédit (debit / credit) — ordre conseillé : initiaux à gauche, débit/crédit période au centre, finaux à droite (glisser-déposer si l’UI le permet).
  3. Balance OHADA (toolbox) : mêmes expression_label préfixés ohada6_ (ohada6_open_deb, … ohada6_close_cre). En v19, la création tente d’abord le moteur aggregation (initial_debit / debit, etc. + subformula: positive sur les colonnes décomposées) — ce qui évite le piège du moteur domain avec [('debit','>',0)] (filtre d’écritures sans séparer correctement débit/crédit sur une ligne groupée). Si le serveur refuse aggregation sur groupby, repli automatique en domain (cumuls bruts sur les colonnes extérieures). Env TOOLBOX_OHADA6_FORCE_DOMAIN=1 force ce repli ; TOOLBOX_OHADA6_OUTER_NET=1 n’a d’effet qu’en mode domain. Rapport existant : boutons Colonnes extérieures → brut et Mode net OHADA — aggregation sur la page staff. La liste ci-dessus reste valable en configuration manuelle si les libellés sont cohérents partout.
  4. Onglet Lignes — ligne principale (souvent « Accounts », « Balance », etc.) avec groupby = account_id. Sous Expressions, créer ou vérifier une expression par colonne (label = expression_label de la colonne) :
    LabelMoteurFormuleSubformula
    debit_initialaggregationinitial_debit - initial_creditpositive
    credit_initialaggregationinitial_credit - initial_debitpositive
    debitaggregationdebit—
    creditaggregationcredit—
    debit_finalaggregation(initial_debit + debit) - (initial_credit + credit)positive
    credit_finalaggregation(initial_credit + credit) - (initial_debit + debit)positive
  5. Onglet Options du rapport : activer Inclure le solde initial (obligatoire sinon initiaux à 0) ; vérifier la période ; désactiver la comparaison de périodes si elle duplique les colonnes.
  6. Pièges fréquents : message « expression inconnue » → le label de l’expression doit être strictement égal à l’expression_label de la colonne. Initiaux à 0 → option solde initial. Tout à zéro → mauvaise ligne (il faut la ligne avec groupby = account_id).

Gabarit XML : le dépôt contient examples/balance_generale_6_col_studio.example.xml. Depuis cette page (connecté staff), vous pouvez le télécharger ici, puis l’importer dans Odoo via Studio → Import XML (ou un module). Alternative sans fichier : exécuter localement python create_balance_6cols_via_api.py (même contenu fonctionnel). La toolbox n’exécute pas l’import sur votre base depuis le navigateur : volontaire (droits, validation Odoo, risques). En cas d’échec Studio, importer en deux temps (colonnes puis ligne + rapport).

Alternative manuelle — 6 colonnes (Odoo v19 / Studio)

Pour une balance générale type OHADA (débit/crédit initiaux et finaux séparés), en v19 : utiliser le moteur aggregation sur les expressions (pas une formule Python libre). Variables usuelles : initial_debit, initial_credit, debit, credit. Ce n’est pas le même schéma que la balance d’essai Enterprise « trial balance handler ». Le détail pas à pas (colonnes, lignes, options, pièges) est dans le bloc replié Procédure manuelle d’ajout des colonnes ci-dessus.

Subformules (v19) — à la place de max(x, 0) :

subformula Effet
positiveGarde uniquement les valeurs > 0 (côté débit affiché).
negativeGarde uniquement les valeurs < 0 (utile pour certains découpages crédit).

Colonnes (6) — schéma type (formule d’agrégation + positive pour les soldes décomposés) :

Colonne formula (aggregation) subformula
Débit initialinitial_debit - initial_creditpositive
Crédit initialinitial_credit - initial_debitpositive
Débit périodedebit—
Crédit périodecredit—
Débit final(initial_debit + debit) - (initial_credit + credit)positive
Crédit final(initial_credit + credit) - (initial_debit + debit)positive

Rappels : libellés d’options selon la langue ; Studio SaaS : mode développeur → Import XML. Si max() n’est pas accepté : (a) if a > 0 else 0. Pour des totaux par classe OHADA (1–7), il faut en général des lignes d’agrégation ou de structure supplémentaires (hors de ce guide).
