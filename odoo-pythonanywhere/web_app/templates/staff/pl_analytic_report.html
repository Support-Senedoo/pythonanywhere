Descriptif — compte de résultat analytique (API)

But. Générer un nouveau tableau réalisé / budget / % par compte général pour l’axe analytique choisi (projet, etc.). Les données viennent de l’API Odoo (account.move.line, analytic_distribution, crossovered.budget.lines). Aucun enregistrement account.report n’est créé pour ce tableau — le calcul est côté API. Vous pouvez en revanche, plus bas, dupliquer et personnaliser un rapport Odoo (comme l’utilitaire « P&L analytique / budget ») : copie, renommage, suppression.

Période : calcul automatique du 1er janvier de l’année en cours jusqu’à la date du jour (pas de saisie manuelle).

Axe analytique : choisissez le libellé dans la liste (recherche par nom ou code). Les montants réalisés suivent analytic_distribution (prorata des %).

CPC SYSCOHADA — Budget Analytique (rapport neuf)

Crée un rapport account.report complet depuis zéro : structure CPC SYSCOHADA, 4 colonnes (Réalisé / Budget / Écart / % Réalisation), filtre analytique activé. Toute instance Senedoo existante est supprimée avant la création.

La colonne Budget utilise le moteur natif engine='budget' (données crossovered.budget.lines). Le rapport est créé avec filter_analytic et, si votre version l’expose, filter_budgets / filter_budget — comme sur une copie P&L Senedoo : dans Odoo, choisissez la période, le budget et l’analytique dans les filtres du rapport.
