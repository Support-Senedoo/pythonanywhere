Mode d’emploi — analytique, budget financier (SaaS) et CPC

Trois leviers complémentaires. (1) Tableau API ci-dessous : réalisé / budget / % calculés sur le serveur toolbox à partir d’Odoo. La colonne budget lit les budgets croisés (crossovered.budget.lines, module Budget classique). Elle ne lit pas les montants des budgets financiers du reporting (account.report.budget). (2) Budget financier + analytique dans l’UI Odoo : bouton « Créer copie P&L pilotage » sur un account.report existant — la copie active filtres analytique et budgets ; vous choisissez le budget financier dans Odoo. (3) CPC SYSCOHADA : wizard installé dans Odoo (Réalisé / Budget / Écart / %, PDF/Excel natifs) avec rattachement budget financier + axe (champs créés par la toolbox sur les budgets).

Si le tableau API affiche un budget vide alors que vous n’utilisez que des budgets financiers : créez ou complétez des lignes budgets croisés sur le même axe, ou passez par la copie P&L / le wizard CPC. Dépannage détaillé : DEPLOY_PYTHONANYWHERE.md (checklist budget × analytique).
