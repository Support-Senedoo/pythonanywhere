Sonde des bases Odoo par compte

But & principe

Trois modes (rien n’est enregistré côté serveur ; le mot de passe reste masqué et n’est pas réaffiché après envoi ; le cookie n’est jamais réaffiché) : (1) laissez l’URL vide : connexion au portail Odoo.com (page « Mes bases »), liste des instances *.odoo.com liées au compte, puis test XML-RPC sur chacune. (1bis) même chose sans mot de passe portail : connectez-vous à la main sur odoo.com (captcha OK), ouvrez « Mes bases », puis collez l’en-tête Cookie de la requête document (voir ci-dessous) — la liste vient du serveur toolbox avec vos cookies ; le mot de passe reste utile pour l’étape XML-RPC sur chaque base. (2) URL d’une instance : db.list() sur cette instance si le service db existe, puis authentification sur chaque base listée. Indications si accès OK : sociétés, module base, utilisateur.

  • Mode portail : dépend du HTML de odoo.com — en cas d’échec, vérifiez langue (variables d’environnement TOOLBOX_ODOO_PORTAL_LANG, TOOLBOX_ODOO_PORTAL_ORIGIN).
  • Mode URL : si db.list() est absent (souvent Odoo SaaS 19+), la liste reste vide — utilisez alors le mode sans URL.
  • Maximum {{ max_probe }} bases testées par requête (sécurité / performance).
  • Ne pas utiliser sur une machine partagée sans confiance : identifiants transmis à odoo.com et aux instances.
  • Si le portail refuse la connexion automatique : captcha / IP datacenter — utilisez le cookie de session après connexion manuelle, ou le mode URL d’instance (si db.list() est disponible). 2FA / SSO : le cookie peut suffire pour « Mes bases » si votre navigateur est déjà connecté.
  • Cookie = session complète : ne le partagez pas ; ne le collez pas dans un chat ou un ticket public.

Session navigateur (captcha / PythonAnywhere)
  1. Même langue que le serveur toolbox si possible (TOOLBOX_ODOO_PORTAL_LANG, ex. /fr_FR).
  2. Dans Chrome ou Edge : F12 → onglet Réseau, cochez « Conserver le journal ».
  3. Ouvrez la page Mes bases sur odoo.com (connecté, captcha déjà validé).
  4. Cliquez la requête du document (type « document » / HTML) vers …/my/databases.
  5. Onglet En-têtes → section En-têtes de requête → copiez la valeur complète de Cookie (une longue ligne).
  6. Collez-la ci-dessous. Le mot de passe portail peut rester vide dans ce cas ; gardez le même e-mail / login que sur odoo.com pour XML-RPC.
